Display, visibilty y opacity

Display

La propiedad display es la encargada de controlar si un elemento se comportará como un elemento block o uno inline (entre otras opciones).

Como ya hemos visto todos los elementos tendrán un comportamiento display por defecto pero con esta podemos sobreescribirlo.

Display inline-block

Los elementos inline tienen una manera particular de interpretar el box-model, por ejemplo los margenes superior e inferior no afectan a los elementos que lo rodean, tampoco aceptan width o height

Si queremos que un elemento inline acepte el box model al completo, pero no genere un salto de línea (como uno block), tendremos que asignarle el valor inline-block.


 p { display: inline-block; }

Display none

El valor none en la propiedad display removerá por completo el elemento de nuestra página web. Los elementos que le sigan en el HTML ocuparán su lugar en el flow del documento.

Un lector de voz ignorará nuestro contenido con display: none, lo cual tenemos que tener en cuenta si no queremos afectar la accesibilidad a la hora de ocultar algo.

Visibility

La propiedad visibility nos permite hacer invisible un objeto pero que este siga ocupando su logar en el flow del documento, por lo que el resto de los elementos seguirán comportándose como si estuviera allí.

Los posibles valores para visibiliy son:
visible | hidden | collapse

Opacity

Podemos controlar la opacidad de un elemento con la propiedad opacity, cuyo valor va de 0 (transparente) a 1 (opaco).

Hay que tener en cuenta que los elementos contenidos por el elemento al que apliquemos opacity también se verán afectados y su opacidad máxima siempre será la del elemento padre.


<p>El parrafo tiene una opacidad del 50% por eso, aunque el strong tenga
el 100%, <strong>sigue siendo semitransparente.</strong></p>
                    

p { opacity: .5; }
strong { opacity: 1; }

El parrafo tiene una opacidad del 50% por eso, aunque el strong tenga el 100%, sigue siendo semitransparente.

Posicionando elementos con float

En CSS hay varias maneras de posicionar los elementos dentro de la página. Una de las más utilizadas es a través de la porpiedad float.

float nos permitie remover a un elemento de su posición natural en el flow de la página y posicionarlo contra uno de los lados de su elemento contenedor. Los elementos que están a continuación suyo se limitarán a rodearlo.

En su origen float se creo para posicionar imágenes dentro de un bloque de texto pero con el tiempo acabo por ser una de las herramientas CSS más usadas para generar layout en nuestras paginas (por falta de otras herramientas mejores).

clear

Si en algun momento queremos que un determinado elemento ignore una propiedad float de un elemento anterior, deberemos recurrir a la propiedad clear.

clear soporta 4 valores:

  • none: el elemento se verá afectado normalmente por un float.
  • left: el elemento se desplazará abajo cuando se encuentre con un float: left.
  • right: el elemento se desplazará abajo cuando se encuentre con un float: right.
  • both: el elemento se desplazará abajo ante cualquier float.

clearfix

Un problema que suele presentarse cuando utilizamos float para crear layouts es que al remover todos los elementos en un contenedor del flow natural el contenedor se queda vacío y su altura pasa a ser 0.

Hay varios técnicas para solucionar este problema. Este conjunto de técnicas se suelen conocer con el nombre de clearfix, y aunque todas se agrupen bajo el mismo nombre sulen diferir mucho unas de otras.

Hay que tener en cuenta que la propiedad float nunca fue diseñada para ser usada como una herramienta de layout por lo que tanto el echo de utilizarla para componer páginas, como las distintas técnicas para solucionar los problemas asociados, no dejan de ser hacks y cada uno acarrea un serie de ventajas e inconvenientes que hay que considerar a la hora de utilizarlos.

Una técnica sencilla para solucionar el problema es colocar dentro del contenedor, a continuación de los elementos flotados, un elemento con la propiedad clear: both

1
2
3
clear: both;

Otra posible solución, es recurrir a la propiedad overflow. Cualquier valor diferente a visible nos corregirá el problema 


div { overflow: auto; }

2
3

Hay otras técnicas más avanzadas y estables de clearfix que combinan varias propiedades. Es una buena idea contar en el css con una class .clearfix de ayuda para poder utilizar cuando tengamos que lidiar con el problema.

Una de las técnicas más populares es la siguiente:


.clearfix:before,
.clearfix:after {
    content: " ";
    display: table;
}
.clearfix:after {
    clear: both;
}
.clearfix {
    *zoom: 1;
}
