Selector descendente

(Descendant selector)

El selector descendiente (descendant selector) actúa sobre un elemento que se encuentra contenido dentro de otro.

Se trata de un selector contextual ya que afecta a un elemento no solo por él mismo, sino también por su relación con otros elementos.

<p>Y esto es un <strong>párrafo</strong> que támbien tiene un strong.

Especificidad

(Specificity)

En CSS, la especificidad, es lo que determina que estilo será el que acabará aplicandose a un elemento determinado.

Cuando a un elemento se le aplica más de un selector con reglas que actúan sobre la misma propiedad, es la especificidad la encargada de resolver este conflicto.

A grandes rasgos podemos ordenar los diferentes tipos de selectores de mayor a menor peso específico de la siguiente manera:

  1. IDs
  2. Clases
  3. Selectores contextuales
  4. Selectores individuales

div y span

HTML nos ofrece dos elementos genéricos para agrupar nuestro contenido, con el fin de poder asignar selectores CSS más específicos: Division (<div>) y Span (<span>).

En el primer caso se trata de un elemento block, mientras que el segundo es un elemento inline

<div> y <span> nos permiten poder asignar clases o ids a diferentes contenidos aunque estos no posean un valor semántico concreto.

De todas maneras, antes de utilizarlos, hay que asegurarse que no exista ningún otro elemento más apropiado para ese contenido.


<div class="ejemplo">
    <h1>Ejemplo de div</h1>
    <p>Podemos usar div para agrupar algunos elementos y luego otorgar
    a ese grupo unos estilos concretos</p>
    <p><span class="capital-letter">S</span>pan por su parte nos permite
    agrupar letras, palabras u otros elementos inline.</p>
</div>

Imágenes

Para agregar imágenes en la web utilizaremos el tag <img>. Se trata de un tag vacío por lo que no lleva ni contenido ni etiqueta de cierre.

La ruta de la imágen a incluir la determinaremos con el atributo src. También podemos proporcionar un texto alternativo que el usuario verá si la imágen no se carga, gracias al atributo alt.

<img src="ruta-de-la-imagen.jpg" alt="texto alternativo">

El elemento imágen es un elemento inline. Por esta razón, si lo utilizamos dentro lo que no provocará un salto de línea sino que se insertará entre el texto provocando que esa line-height de esa línea pase a ser la altura de la imágen.

Formatos de imágenes

No todos los formatos de imágenes estan disponibles para ser usados en web.

Los formatos de imágen en mapa de bits permitidos para web son JPG, GIF y PNG.

GIF

GIF es el formato web más antiguo. Su compresión se realiza sin perdida de calidad en la imágen pero está limitado a una paleta de 256 colores. Posee un canal de transparencia pero por lo limitado de su paleta no da muy buen resultado sobre todo con superficies curvas.

Es el único de los 3 formatos que permite animación. Es decir, incluir varias imágenes dentro de un mismo archivo que se irán sucediendo en el tiempo.

JPG

JPG es un formato de 24-bits, capaz de mostrar millones de colores. Esto lo convierte en ideal para fotografía o imágenes con transiciones de color suaves.

Su algoritmo de compresión acarrea perdida de calidad por lo que, a la hora de generar una imágen jpg, hay que evaluar la relación calidad-peso que queremos conseguir.

PNG

PNG es el formato más nuevo de los 3. Algunas de ñas características que lo hacen destacar son:

  • Puede contener imágenes indexadas de 8-bit, RGB 24-bit, 16-bit en escala de grises, e incluso de color 48-bit.
  • Compresión sin pérdida de calidad.
  • Varios niveles de transparencia.
  • Texto embebido con información acerca del autor, copyright, etc.

Listas

HTML cuenta con 3 tipos de listas:

  • Unordered lists: Un listado de elementos sin un orden concreto.
  • Ordered lists: Un listado de elementos dónde el orden es importante.
  • Description lists: Un listado compuesto de pares término/descripción.

Unordered Lists (<ul>)

Marcaremos una lista dónde el orden de los elementos es indiferente, con el tag <ul>.

A diferencia de otros elementos que funcionan solos, <ul> necesita un segundo elemento que nos servirá para marcar cada item de la list: list item (<li>).


<ul>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
    <li>Item 4</li>
</ul>
  • Item 1
  • Item 2
  • Item 3
  • Item 4

Ordered Lists (<ol>)

Si el orden de los elementos de nuestra lista es relevante, como por ejemplo en una receta de cocina o en un top-ten, usaremos tag <ol>.

Para marcar los items de la lista también será necesario el tag <li>.


<ol>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
    <li>Item 4</li>
</ol>
  1. Item 1
  2. Item 2
  3. Item 3
  4. Item 4

Description lists (<dl>)

La estructura de las definition list (<dl>) es un poco diferente a la de las dos listas anteriores.

En lugar de items una <dl> está compuesta por una suseción de términos (<dt>) con sus respectivas definiciones(<dd>).

Un <dt> puede estar seguido de múltiples <dd> pero no a la inversa.


<dl>
    <dt>Title</dt>
    <dd>Back to the Future</dd>
    <dt>Director</dt>
    <dd>Robert Zemeckis</dd>
    <dt>Stars</dt>
    <dd>Michael J. Fox</dd>
    <dd>Christopher Lloyd</dd>
</dl>
